Exploring Interfaces in Java

Welcome to our comprehensive guide on interfaces in Java! In this tutorial, we'll delve into the world of interfaces, a powerful tool for achieving abstraction and polymorphism in Java programming.

Understanding Interfaces

An interface in Java is a reference type, similar to a class, that can contain only constants, method signatures, default methods, static methods, and nested types. It provides a way to achieve abstraction by defining a contract for what classes can do without specifying how they do it.

Declaring Interfaces

In Java, you declare an interface using the interface keyword, followed by the interface name and a code block containing method signatures:

interface MyInterface {
        void method1();
        int method2(String str);
    }

An interface can extend one or more other interfaces, allowing you to inherit method signatures from multiple sources:

interface MyInterface extends AnotherInterface, YetAnotherInterface {
        // Method signatures
    }

Implementing Interfaces

To use an interface in a class, you must implement it using the implements keyword. When a class implements an interface, it must provide concrete implementations for all the methods declared in the interface:

class MyClass implements MyInterface {
        // Implement interface methods
    }

Interface Default Methods

Java 8 introduced default methods in interfaces, allowing you to add new functionality to interfaces without breaking existing implementations. Default methods have a default implementation specified in the interface itself:

interface MyInterface {
        void method1();
    
        default void defaultMethod() {
            // Default implementation
        }
    }

Interface Static Methods

Java 8 also introduced static methods in interfaces, which can be called using the interface name without the need for an instance of the interface. Static methods provide utility methods related to the interface:

interface MyInterface {
        void method1();
    
        static void staticMethod() {
            // Static method implementation
        }
    }

Let's practise your understanding about Interfaces

Java code that demonstrates the declaration and initialization of Interfaces: 

interface Vehicle {
    void start();
    void stop();
}

class Car implements Vehicle {
    @Override
    public void start() {
        System.out.println("Car started.");
    }

    @Override
    public void stop() {
        System.out.println("Car stopped.");
    }
}

class Bike implements Vehicle {
    @Override
    public void start() {
        System.out.println("Bike started.");
    }

    @Override
    public void stop() {
        System.out.println("Bike stopped.");
    }
}

public class Main {
    public static void main(String[] args) {
        Car car = new Car();
        Bike bike = new Bike();

        car.start();
        car.stop();

        bike.start();
        bike.stop();
    }
}

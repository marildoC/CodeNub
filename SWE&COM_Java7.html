Understanding Polymorphism in Java

Welcome to our comprehensive guide on polymorphism in Java! In this tutorial, we'll explore the concept of polymorphism, its significance in object-oriented programming, and how it can be implemented in Java.

What is Polymorphism?

Polymorphism, derived from the Greek words "poly" (many) and "morphos" (forms), refers to the ability of an object to take on many forms. In the context of Java, polymorphism allows objects of different classes to be treated as objects of a common superclass.

There are two main types of polymorphism in Java:

  • Compile-time Polymorphism: Also known as method overloading, compile-time polymorphism occurs when multiple methods in the same class have the same name but different parameters.
  • Runtime Polymorphism: Also known as method overriding, runtime polymorphism occurs when a subclass provides a specific implementation of a method that is already defined in its superclass.

Compile-time Polymorphism (Method Overloading)

In Java, method overloading allows us to define multiple methods with the same name but different parameter lists within the same class. The compiler determines which method to call based on the number and types of arguments passed to the method.

Let's look at an example of method overloading:

        class MathOperations {
            int add(int a, int b) {
                return a + b;
            }
    
            double add(double a, double b) {
                return a + b;
            }
        }
    
        public class Main {
            public static void main(String[] args) {
                MathOperations math = new MathOperations();
                System.out.println(math.add(5, 3));
                System.out.println(math.add(2.5, 3.7));
            }
        }

In this example, the MathOperations class defines two add methods: one that accepts two int parameters and another that accepts two double parameters. The appropriate method is called based on the argument types provided.

Runtime Polymorphism (Method Overriding)

Method overriding allows a subclass to provide a specific implementation of a method that is already defined in its superclass. This enables dynamic method invocation, where the method to be executed is determined at runtime based on the actual object type.

Let's see an example of method overriding:

        class Animal {
            void makeSound() {
                System.out.println("Some sound");
            }
        }
    
        class Dog extends Animal {
            void makeSound() {
                System.out.println("Bark");
            }
        }
    
        public class Main {
            public static void main(String[] args) {
                Animal animal = new Dog();
                animal.makeSound();
            }
        }

In this example, the Dog class overrides the makeSound method defined in the Animal class. When we create a Dog object and call the makeSound method, the overridden method in the Dog class is invoked instead of the one in the Animal class.

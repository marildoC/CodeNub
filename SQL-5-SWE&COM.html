Introduction to SQL Joins

SQL joins are a fundamental aspect of relational databases, allowing you to merge data from different tables based on specified conditions. Understanding SQL joins is essential for anyone working with relational databases.

Inner Join

The INNER JOIN clause is used to combine rows from two or more tables based on a related column between them. It returns only the rows that have matching values in both tables.

Left Join

The LEFT JOIN (or LEFT OUTER JOIN) clause returns all rows from the left table and the matched rows from the right table. If there are no matches, NULL values are included for the columns from the right table.

Right Join

The RIGHT JOIN (or RIGHT OUTER JOIN) clause returns all rows from the right table and the matched rows from the left table. If there are no matches, NULL values are included for the columns from the left table.

Full Join

The FULL JOIN (or FULL OUTER JOIN) clause returns all rows when there is a match in either the left or right table. If there are no matches, NULL values are included for the columns from the table without a match.

Cross Join

The CROSS JOIN clause returns the Cartesian product of the two tables, i.e., all possible combinations of rows from both tables.

Let's try one SQL example

-- Create a simple database schema
            CREATE DATABASE School;
            
            -- Create a table to store student information
            CREATE TABLE Students (
                StudentID INT PRIMARY KEY,
                FirstName VARCHAR(50),
                LastName VARCHAR(50),
                Age INT,
                GPA FLOAT
            );
            
            -- Create a table to store department information
            CREATE TABLE Departments (
                DepartmentID INT PRIMARY KEY,
                DepartmentName VARCHAR(50)
            );
            
            -- Create a table to associate students with departments
            CREATE TABLE StudentDepartments (
                StudentID INT,
                DepartmentID INT,
                FOREIGN KEY (StudentID) REFERENCES Students(StudentID),
                FOREIGN KEY (DepartmentID) REFERENCES Departments(DepartmentID)
            );
            
            -- Insert some sample data into the Students table
            INSERT INTO Students (StudentID, FirstName, LastName, Age, GPA)
            VALUES 
                (1, 'John', 'Doe', 20, 3.5),
                (2, 'Jane', 'Smith', 22, 3.8),
                (3, 'Michael', 'Johnson', 21, 3.2);
            
            -- Insert some sample data into the Departments table
            INSERT INTO Departments (DepartmentID, DepartmentName)
            VALUES 
                (1, 'Engineering'),
                (2, 'Marketing'),
                (3, 'Human Resources');
            
            -- Associate students with departments
            INSERT INTO StudentDepartments (StudentID, DepartmentID)
            VALUES
                (1, 1), -- John Doe is in Engineering
                (2, 2), -- Jane Smith is in Marketing
                (3, 1); -- Michael Johnson is also in Engineering
            
            -- Retrieve all students along with their department names using INNER JOIN
            SELECT s.FirstName, s.LastName, s.Age, s.GPA, d.DepartmentName
            FROM Students s
            INNER JOIN StudentDepartments sd ON s.StudentID = sd.StudentID
            INNER JOIN Departments d ON sd.DepartmentID = d.DepartmentID;
